# Implementation Decisions Log

Decisions made during Phase 1B-1 implementation that were not covered by the PRD. These supplement the PRD's own Decision Log (Section 13).

| # | Date | Decision | Rationale | Status |
|---|---|---|---|---|
| D-IMPL-001 | 2026-04-02 | Default artifact format for novels/non-fiction/Hindi books = `Paperback` | No column in the spreadsheet distinguishes hardcover from paperback. Paperback is the more common format. Users can update individually. Approved by product owner during planning. | Approved |
| D-IMPL-002 | 2026-04-02 | Entity-level field provenance (`field_name="*"`, `source="migrated"`) rather than per-field | Creating a provenance record for every non-null field would generate 10K+ records all saying "migrated" with no differentiation. One record per entity marks it as spreadsheet-sourced. Per-field provenance is reserved for Phase 2+ scraped data where the source varies per field. Approved by product owner during planning. | Approved |
| D-IMPL-003 | 2026-04-02 | Work title for reprinted comics = `"{Original Volume} #{O#}"` | E.g., "Batman #455" not "Batman #7 (Gotham)". The Work represents the original story, so its title should reference the original publication. The Artifact title captures the reprint identity. | Approved |
| D-IMPL-004 | 2026-04-02 | Work deduplication key for comic Works = `(volume_run_id, issue_number)` | A comic Work could be created from both Comics Issues and Comics GNs sheets (if a GN collects an issue that also exists individually). This key prevents duplicates while allowing distinct Works from different volume runs. | Approved |
| D-IMPL-005 | 2026-04-02 | Creator dedup by normalized `sort_name` (canonical "Last, First" form, case-insensitive) | More robust than display_name dedup. "Grant, Alan" and "Alan Grant" resolve to the same normalized key. Near-duplicates (same last name, different first name) are flagged as `name_inconsistency` data quality flags rather than silently merged. | Approved |
| D-IMPL-006 | 2026-04-02 | Story arc deduplication by `(name, total_parts)` | Two arcs named "Identity Crisis" exist — one with 2 parts (Batman) and one with 7 parts (DC crossover event). Using name alone would incorrectly merge them. | Approved |
| D-IMPL-007 | 2026-04-02 | "Various" treated as a real Creator record | Per PRD edge case guidance. Creating a Creator with display_name="Various" is simpler than special-casing it. Users can resolve via the Review Queue if desired. | Approved |
| D-IMPL-008 | 2026-04-02 | Asterisk in O#/issue number = `is_partial` flag on ArtifactWork | Batman #22's O2# value is "18*" indicating partial content. The asterisk is stripped for the issue number, and `ArtifactWork.is_partial = True` is set. This matches PRD Section 7.4 edge case #1. | Approved |
| D-IMPL-009 | 2026-04-02 | Idempotency = drop-and-recreate all tables on each migration run | This is a one-shot migration script, not an incremental sync. Clean-slate approach ensures re-running after fixing a parse bug always produces consistent results. | Approved |
| D-IMPL-010 | 2026-04-02 | Added `source_sheet` column to artifacts table | Not in PRD schema but essential for verification: tracks which xlsx sheet each artifact was migrated from. Enables independent row count verification per sheet. Small addition, high value. | Flag for review |
| D-IMPL-011 | 2026-04-02 | For GNs with Original Volume containing `;` (semicolon), the original volume name is treated as a single work | The DC Elseworlds case has `"Green Lantern: 1001 Emerald Nights; Superman: Last Stand on Krypton"` in Original Volume. The Collects field is what determines individual Works; the Original Volume field is used for the volume run. Complex cases are flagged for manual review. | Approved |
| D-IMPL-012 | 2026-04-02 | Collects parsing uses conservative regex: only auto-parses simple `"Title #X-Y"` patterns | Per PRD Decision D-015. Complex Collects text (partial issues, mixed formats, multiple series) is flagged as `unparsed_collects` for manual review. 50 of 96 GNs were auto-parsed, 12 flagged, 27 had special values (-, DNF, etc.) skipped. | Approved |
| D-IMPL-013 | 2026-04-02 | DC Elseworlds: Original Volume with `;` treated as 2 separate Works (overrides D-IMPL-011) | Red Teamer + owner confirmed "Green Lantern: 1001 Emerald Nights; Superman: Last Stand on Krypton" are two distinct one-shots. Creator roles split accordingly (LaBan/Guay → GL, Gerber/Wheatley → Superman). D-IMPL-011's single-work treatment was incorrect for this case. | Approved |
| D-IMPL-014 | 2026-04-02 | completion_status schema fix deferred to PRD update | Owner clarified Y/N is per-arc, not per-volume-run. Requires adding completion_status to story_arcs table and re-deriving from xlsx. Too structural for a DB patch — needs PRD Section 5.2 + 6.3.6 update, then re-migration. See PRD Decision D-028. | ~~Approved — deferred~~ Implemented in v0.2.0 |
| D-IMPL-015 | 2026-04-02 | Applied D-028 as targeted DB patch (ALTER TABLE + re-derive) instead of full re-migration | PRD v1.1 provided the corrected schema. Patch approach: ALTER TABLE to add column, re-derive from xlsx, clear volume_runs. Avoids re-migration risk. 4 arc names unmatched due to comma/semicolon parsing differences in original migration — left for manual resolution. | Approved |
| D-IMPL-016 | 2026-04-03 | Cover image format: valid JPEG only, ≥1KB, reject known placeholder hashes | PRD §10.4 does not specify image dimensions or format. JPEG is the universal output of all three sources (Open Library, Google Books, ComicVine). Minimum size 1KB catches OL's 1×1 placeholder. Google Books additionally returns a 9,103-byte uniform grey "no preview" image (MD5 `a64fa89d7ebc97075c1d363fc5fea71f`) — this is detected by hash and rejected. No dimension normalisation is applied at scrape time; the UI (Phase 2) can resize on display. | Approved |
| D-IMPL-017 | 2026-04-03 | Cover scraping source priority: Open Library direct ISBN → Google Books ISBN → Open Library Search API (via cover_i) → (GNs only) Google Books title+author | Open Library direct is fastest and highest quality. Google Books ISBN covers occasional OL gaps but returns a fixed placeholder for unknowns (rejected by hash). OL Search API finds covers via Work entity even when the ISBN link is missing in OL's index — added as third source after observing ~30 additional rescues (e.g. Hercule Poirot's Christmas, The Hobbit, Never Go Back). | Approved |
| D-IMPL-018 | 2026-04-03 | ComicVine web scraping replaced by ComicVine free API | ComicVine's website uses Cloudflare anti-bot protection, making scraping infeasible from a server environment. The official free API (register at comicvine.gamespot.com/api) bypasses this entirely and returns structured JSON with `super_url` cover images. API key stored in `.env`, not committed. | Approved |
| D-IMPL-019 | 2026-04-03 | ComicVine API delay kept at 1.0s per call | CV documents 200 req/hour per endpoint. At 1s delay, sequential scraping reaches ~180 API calls/hour per endpoint type (/api/issue/ and /api/search/ counted separately). CDN image downloads do not count against the API quota. Parallelisation would risk exceeding the limit and is not warranted for a one-time scrape. | Approved |
